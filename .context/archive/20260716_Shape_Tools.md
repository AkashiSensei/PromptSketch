# Shape Tools

Status: COMPLETED on 2026-07-16

## Goal

为 PromptSketch 增加矩形、圆形 / 椭圆和圆角矩形工具，并将颜色能力抽离为画笔与形状共享的主题感知调色板。

## Issue Reference

- 无外部 Issue；需求来自当前任务对话。

## Implementation Details

- 在现有 Brush 与 Stroke Eraser 之外增加 Shape 工具，并提供 Rectangle、Ellipse、Rounded rectangle 三种形状类型。
- 采用按下确定起点、拖动实时预览、抬起提交的交互；支持从任意方向拖动并正确归一化边界。
- Ellipse 默认允许任意宽高；按住 `Shift` 时约束为圆。Rectangle 与 Rounded rectangle 按住 `Shift` 时约束为正方形。
- 圆角矩形使用固定的视觉圆角，并按形状短边自动收敛，避免小尺寸形状出现非法或畸形圆角。
- 形状配置包含线框粗细和“线框 / 实心”开关；线框模式只描边，实心模式只使用当前颜色填充且不绘制边框。
- 将颜色从 `BrushSettings` 专属概念中抽离为共享、主题感知的颜色模型与调色板：
  - 画笔与形状共用唯一的当前颜色，不区分描边色和填充色。
  - 当前颜色从现有 Ink / Red / Blue / Green / Yellow 颜色槽中选择，并继续支持颜色槽自定义、明暗主题配对和本地持久化。
  - 已提交的笔画和形状保存创建时的完整颜色对，之后切换当前颜色不会修改历史标注；切换明暗主题时使用各自保存的主题颜色。
- 颜色面板固定在控制面板顶部并保持稳定高度；Stroke Eraser 模式只将其置灰禁用，不从布局中移除，避免工具切换造成颜色控件跳动。
- 将当前只保存 `Stroke[]` 的模型扩展为按绘制顺序保存的标注联合类型（stroke / shape），统一负责重绘、清空和后续历史能力。
- Stroke Eraser 命中形状的填充区域或可见边框时删除整个形状，保持“整形擦除”语义；底图仍不受影响。
- 切换工具或取消指针时应清理未完成的预览；零尺寸拖动不提交无意义形状。
- 本任务不扩展直线、选中 / 移动 / 缩放形状、独立圆角半径、填充透明度、撤销系统或项目持久化，以保持轻量范围。

## Test Plan

- Build：运行 `npm run build`，确保 TypeScript 与 Vite 构建通过。
- Geometry unit coverage：为边界归一化、Shift 等比约束、圆角钳制以及矩形 / 椭圆 / 圆角矩形的橡皮命中补充可重复测试；若项目暂未配置测试框架，优先把几何逻辑提取为纯函数并至少通过 TypeScript 构建与针对性脚本验证。
- UI / manual：逐一创建三种形状，验证八个拖动方向、实时预览、短距离 / 零尺寸拖动、工具切换与指针取消。
- Styling：分别验证线框和纯实心模式、线框粗细、共享颜色选择以及自定义颜色槽；确认实心模式没有边框。
- Theme：在亮 / 暗主题间切换，验证既有笔画与形状使用各自保存的主题色，当前颜色控件同步正确。
- Eraser：从形状内部、边框以及形状外部划过，验证只删除命中的完整标注且不破坏背景；同时回归自由笔画擦除。
- Regression：回归画笔压感、画笔 / 橡皮尺寸滚轮调整、画布平移缩放、新建画布和清空标注。
- Responsive / accessibility：检查窄屏面板布局、工具按钮选中状态、标签、键盘焦点和控件可读名称。

## Focusing Files

- `src/colors.ts`（新增）：共享颜色类型、颜色槽、主题配对与持久化边界。
- `src/canvas.ts`：形状与统一标注模型、实时预览、渲染及整形擦除命中。
- `src/geometry.ts`：形状边界、等比约束与命中检测纯函数。
- `src/main.ts`：工具 / 形状 / 颜色状态和控制面板事件编排。
- `src/styles.css`：三工具布局、形状配置与独立颜色面板样式。

## Technical Context

- 技术栈保持 TypeScript + Vite，不引入前端框架或后端。
- 产品定位是快速视觉提示与标注，不扩张成设计工具或完整绘画应用。
- 标注层必须与背景 / 粘贴图层分离；任何擦除行为都不能损坏背景。
- 当前颜色槽已经支持 light / dark 成对颜色及 localStorage 覆盖，重构必须兼容现有 `promptsketch.color-overrides` 数据。
- 当前画布按设备像素比渲染、按逻辑坐标保存笔画，并在主题或尺寸变化时完整重绘；形状沿用同一坐标和重绘策略。
- 当前仓库尚无自动化测试命令，不能为了本功能引入重量级测试栈。
