# Line Shape Tool

Status: COMPLETED on 2026-07-22

## Goal

为 PromptSketch 增加可实时预览、可整形擦除并支持 45° 吸附的直线形状工具。

## Issue Reference

- 无外部 Issue；需求来自当前任务对话。

## Implementation Details

- 在现有 Shape 工具中增加 `Line` 类型，并分配未占用的 `L` 快捷键；同步下拉选项、工具按钮快捷键提示、面板收起态和无障碍属性。
- 采用按下确定起点、拖动实时预览、抬起提交的现有形状交互；直线允许任意方向，按住 `Shift` 时以起点为锚点吸附到最接近的 45° 方向，同时保留当前拖动长度。
- 直线保存有方向的起点与终点，不能只保存归一化包围盒；以类型安全的几何表示区分开放线段与现有封闭形状，避免无法区分 `/`、`\` 方向。
- 直线始终按描边渲染，使用共享当前颜色和 Shape 的边框宽度，并使用圆形端帽；它不支持“实心填充”。选中直线时禁用实心开关并清晰表达不可用，但不修改用户原有的填充选择，切回封闭形状后恢复原状态。
- 水平线和垂直线必须可以正常提交；直线使用长度阈值过滤无意义点击，不能复用要求宽高都非零的封闭形状提交条件。
- 实时预览、提交、取消、工具切换、主题重绘、新建 / 清空、撤销 / 重做均沿用现有 Annotation 与历史事务语义。
- Stroke Eraser 在擦除轨迹进入“橡皮半径 + 直线半宽”的范围时删除整条直线；不得影响底图或背景。
- 本任务不包含箭头、折线 / 曲线、端点编辑、选择 / 移动、虚线样式，也不实现自由笔画停顿后的自动拉直。

## Test Plan

- Build：运行 `npm run build`，确保 TypeScript 与 Vite 构建通过，不引入新的前端框架或重量级测试依赖。
- Geometry：针对任意角度端点、水平 / 垂直线、八个拖动方向、45° 吸附、接近角度边界、零长度和最短提交阈值验证纯几何逻辑。
- Rendering：验证实时预览与提交结果一致、DPR / 缩放下端点和线宽稳定、圆形端帽正确，并在亮 / 暗主题下使用创建时保存的主题颜色对。
- UI / shortcuts：验证下拉框和 `L` 均可选择直线，Shape 按钮及收起面板显示正确快捷键；直线状态下实心开关不可用，切回其他形状后原填充选择仍保留。
- Eraser：从直线中段、两端和快速扫过路径进行命中测试，验证只删除命中的整条直线；同时回归封闭形状和自由笔画的整形擦除。
- History / cancellation：验证单次直线提交只产生一个历史事务，撤销 / 重做顺序正确；零长度点击、`pointercancel` 和绘制中切换工具均不留下标注或预览。
- Regression：回归矩形、椭圆、圆角矩形的 Shift 等比约束、线框 / 实心渲染、颜色、画布平移缩放、新建、清空、复制和保存。

## Focusing Files

- `src/geometry.ts`：直线端点、45° 吸附、最短长度与线段命中几何。
- `src/canvas.ts`：开放 / 封闭形状标注模型、直线预览提交、渲染、历史和擦除。
- `src/main.ts`：Line 选项、直线状态下的控件行为、快捷键与无障碍状态同步。
- `src/shortcuts.ts`：新增 `L` 形状快捷键及类型覆盖。
- `src/styles.css`：直线选中时实心控件的稳定禁用态（如现有样式不足）。

## Technical Context

- 技术栈保持 TypeScript + Vite、无前端框架、静态部署，不引入后端。
- 产品定位是快速视觉提示与轻量标注，不扩张为完整绘画、白板或设计工具。
- SPEC 已明确把 line 列入简单形状范围；共享当前颜色和 Shape 线框宽度应继续复用现有设置。
- 标注层与背景 / 粘贴图层分离；整形擦除只删除 Annotation，绝不能损坏背景。
- 现有封闭形状使用归一化 `ShapeBounds`，而直线必须保留起点和终点方向；这是本任务需要显式解决的模型差异。
- 当前仓库没有自动化测试命令；优先保持几何逻辑为纯函数，并以构建和针对性验证覆盖风险。
